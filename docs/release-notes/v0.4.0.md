# AgentPrey v0.4.0

This catch-up release publishes the post-MCP security work that already landed on `master`: initial `tool-misuse` and `approval-bypass` foundations for tool-using agents, plus stronger report surfacing.

## Highlights

- Added deterministic OpenClaw `tool-misuse` findings for dangerous tool + egress combinations and unsafe fallback guidance
- Added deterministic `approval-bypass` findings for risky defaults, prompt-guided bypass, and fallback-path approval avoidance
- Added remote-fallback escalation coverage and improved HTML report surfacing for higher-risk findings

## Install or upgrade

```bash
curl -fsSL https://agentprey.com/install | sh
```

## Install with Cargo

```bash
cargo install agentprey --locked --force
```

## Roll back to this exact version

```bash
cargo install agentprey --locked --version 0.4.0 --force
```

## What's new

### Tool misuse foundation

AgentPrey now ships an initial OpenClaw-focused `tool-misuse` category aimed at release-gating dangerous action patterns in tool-using agents. The shipped rules cover:

- dangerous tool + egress combinations
- unsafe fallback prompt guidance
- unsafe default execution policy
- unsafe remote-fallback escalation

### Approval bypass foundation

AgentPrey now also ships an `approval-bypass` category for approval-sensitive security findings. The current release includes deterministic slices for:

- approval-disabled risky defaults
- prompt-guided approval bypass
- fallback path approval avoidance

Together, these give the CLI a much stronger local-first story for teams that need to gate releases of MCP-connected or tool-using agents.

### Stronger report surfacing

HTML reports now surface higher-risk findings and remediation details more clearly, making it easier to scan results and understand what should block a release.

## Verify

```bash
agentprey --version

```

## Notes

- MCP support remains descriptor-based rather than live-session execution.
- Upload/share smoke remains opt-in during release validation unless `AGENTPREY_UPLOAD_SMOKE=1` is set.
- The shell installer currently supports Linux x86_64 and Apple Silicon macOS.
