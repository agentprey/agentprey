# AgentPrey v0.2.0

This release expands AgentPrey's shipped scan surface with MCP descriptor
security analysis and hardens the initial MCP rules to stay low-noise.

## Highlights

- Added MCP descriptor scanning with `agentprey scan --type mcp`
- Added MCP inventory metadata and additive evidence fields in scan artifacts
- Tightened MCP approval-gap and promptability checks to focus on exact, lower-noise signals

## Install or upgrade

```bash
curl -fsSL https://agentprey.com/install | sh
```

## Install with Cargo

```bash
cargo install agentprey --locked --force
```

## Roll back to this exact version

```bash
cargo install agentprey --locked --version 0.2.0 --force
```

## What's new

### MCP security v1

AgentPrey now supports MCP descriptor scans against local descriptor files. The
initial `mcp-security` ruleset evaluates:

- dangerous standalone capabilities
- risky capability combinations
- remote trust-boundary exposure
- dangerous-tool approval gaps
- dangerous tool-description promptability

### Better downstream evidence

MCP findings now carry richer additive evidence in `agentprey.scan.v1`, while
preserving compatibility for downstream consumers that rely on the existing
schema.

### Lower-noise MCP checks

The MCP promptability rule now avoids partial-phrase matches and ignores
non-dangerous tools even if they contain prompt or policy-related text. The
approval-gap rule similarly stays focused on high-confidence dangerous
capabilities.

## Notes

- MCP support is currently descriptor-based rather than live-session execution.
- Upload/share smoke remains opt-in during release validation.
- The shell installer currently supports Linux x86_64 and Apple Silicon macOS.
