# AgentPrey v0.1.5

This release adds the first public control-center flow while keeping the
existing `scan` command stable for CI/CD and one-shot execution.

## Highlights

- Added `agentprey center` for interactive scan configuration and launch
- Kept `agentprey scan` as the explicit automation and direct-run command
- Locked and documented the `agentprey.scan.v1` downstream artifact contract
- Preserved upload/share behavior and existing scan exit-code semantics

## Install or upgrade

```bash
cargo install agentprey --locked --force
```

## Roll back to this exact version

```bash
cargo install agentprey --locked --version 0.1.5 --force
```

## Verify

```bash
agentprey --version
agentprey center --help
agentprey scan --help
```

## Notes

- `agentprey center` is interactive-only and errors cleanly on non-TTY output.
- `agentprey scan` still supports plain output for CI/CD and `--ui tui` for direct operator use.
- Upload is still opt-in with `--upload`; no telemetry is sent when upload is omitted.
- Share pages remain public-by-link and artifact-driven. There is still no browser dashboard or public bench mode in the shipped product.
