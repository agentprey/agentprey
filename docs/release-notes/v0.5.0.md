# AgentPrey v0.5.0

This release packages the next CLI-facing workflow layer after v0.4.0: scan artifact comparison plus a stronger operator-console experience for interactive runs.

## Highlights

- Added scan artifact comparison foundations with a stable comparison model
- Added `agentprey compare` with JSON and HTML outputs for before/after scan analysis
- Overhauled the operator console TUI for clearer scan progress, completion framing, and control-center flow
- Hardened TUI completion reporting so local artifacts are only shown as written after successful finalization

## Install or upgrade

```bash
curl -fsSL https://agentprey.com/install | sh
```

## Install with Cargo

```bash
cargo install agentprey --locked --force
```

## Roll back to this exact version

```bash
cargo install agentprey --locked --version 0.5.0 --force
```

## What’s new

### Compare command and reports

AgentPrey now supports direct comparison of scan artifacts so teams can inspect whether a target improved, regressed, or shifted in risk between runs. The CLI ships:

- a stable comparison model for downstream tooling
- `agentprey compare` for local artifact-to-artifact analysis
- JSON output for automation pipelines
- HTML compare reports for human review

### Stronger operator console

The TUI now presents a more opinionated operator-console layout for both scan execution and the interactive control center. The release improves panel hierarchy, status framing, and completion readability so a single run is easier to interpret under time pressure.

### Finalization reporting hardening

The control-center completion screen now reports JSON and HTML outputs based on successful writes rather than configured paths alone. If finalization fails partway through, the TUI still shows the local scan outcome while making the output failure explicit.

## Verify

```bash
agentprey --version
agentprey scan --help
agentprey compare --help
agentprey center
```

## Notes

- MCP support remains descriptor-based rather than live-session execution.
- Upload/share smoke remains opt-in during release validation unless `AGENTPREY_UPLOAD_SMOKE=1` is set.
- The shell installer currently supports Linux x86_64 and Apple Silicon macOS.
